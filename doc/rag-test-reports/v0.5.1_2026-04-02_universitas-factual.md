## 5. 분석 및 개선 권고

### 회귀 없음 확인
- content_hash dedup 도입 후 검색 품질 **저하 없음** (v0.5.0 대비 모든 지표 개선).
- Gopedia 아키텍처 도메인 **100% hit** — 자체 문서 검색 신뢰성 높음.

### Miss 패턴

1. **Docker Registry (20%)**: 인증, 이미지 버전, push 절차 등 절차형 사실이 top-5 밖으로 밀림. 청크 내 구체적 버전·URL이 분산되어 있을 가능성.
2. **Traefik 세부 설정 (dashboard_port, docker_sock, three_networks, port_mapping, cf_token)**: 상위 레벨 Traefik 문서는 hit하지만 세부 설정 청크가 순위에서 누락. 청크 granularity 조정 검토 필요.
3. **Docker Network 세부 (reserved_ips, driver/parent, host IP)**: macvlan 생성 명령 요약 페이지는 hit하지만 세부 aux-address/parent 정보가 별도 청크에 있을 때 누락됨.
4. **Server OS (kernel, support_end)**: 커널 버전, EOL 날짜 등 단발성 사실은 임베딩 유사도만으로 검색하기 어려움. BM25 hybrid 강화 또는 metadata 필터 검토.

### 권고사항

| 우선순위 | 권고 | 예상 효과 |
|---------|------|----------|
| HIGH | Traefik references 문서에 포트 매핑·소켓·CF_TOKEN 섹션 분리 명시 | Traefik miss 5개 → hit 전환 |
| HIGH | Docker Registry references에 이미지 버전·인증·push 절차 bullet 추가 | Registry miss 4개 감소 |
| MED | Docker Network aux-address/parent 전용 청크 추가 | Network miss 3개 감소 |
| MED | Server OS 커널 버전·EOL 날짜 명시적 fact 문서 추가 | OS miss 2개 감소 |
| LOW | BM25 hybrid 가중치 튜닝 (숫자·버전 쿼리) | 전반적 precision 개선 |
