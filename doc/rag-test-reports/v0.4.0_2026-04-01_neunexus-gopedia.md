## 5. MRR 하락 원인 분석

### 5-1. 컨텍스트 임베딩의 부작용: 일반적 헤딩

| 쿼리 | 타깃 L3 헤딩 | 문제 |
|------|-------------|------|
| q_server_os_specs | `## Quick reference` | 동일 헤딩이 여러 문서에 존재 → 임베딩 noise |
| q_docker_registry_auth | `## Quick reference` | 상동 |
| q_port_forward_case_ab | `## Case A / B` | 헤딩이 구체적이나 순위가 하락 |
| q_gopedia_smart_sink_routing | `## Smart Sink` | 헤딩 추가로 오히려 경쟁 청크와 유사도 증가 |

**공통 원인**: `## Quick reference`, `## Skill` 등 여러 문서에 동일하게 나타나는 일반적 헤딩을 prepend하면 서로 다른 내용의 청크가 유사한 임베딩을 공유하게 되어 순위 정밀도(ranking precision)가 감소.

**개선 방향 (v0.5.0)**: 헤딩 단일 라인 대신 `L2.summary`(문서 제목 + 섹션 경로 포함)를 컨텍스트로 사용:
```
Before: "## Quick reference\n- **Distro**: Rocky Linux 9.6..."
After:  "Server OS (osteon) Skill > Quick reference\n- **Distro**: Rocky Linux 9.6..."
```

### 5-2. 지속 MISS 쿼리 원인

| 쿼리 | 원인 | 세부 |
|------|------|------|
| q_server_os_specs | **Semantic mismatch** | 한국어 쿼리 vs. 영어 bullet-point. 헤딩 컨텍스트가 `## Quick reference`로 너무 일반적. Rank-1에는 osteon 개요 청크 도달 |
| q_docker_registry_auth | **Semantic mismatch** | 상동. Registry 스펙 bullet-point가 일반 osteon 개요에 밀림 |
| q_gopedia_envelope_strategy | **qrel target mismatch** | 타깃 청크(`GopediaEnvelope` 구조 정의)보다 Consumer 라우팅 동작 청크(rank 2)가 쿼리에 더 적합. qrel 수정 필요 |
| q_infisical_secret_create | **Fragment issue** | `POST /api/v1/secret/secrets` 코드 블록이 여러 L3로 분산. 타깃 청크가 top-10 밖 |

## 7. 재현 방법

```bash
# 1. 브랜치 체크아웃
git checkout feat/v0.4.0-contextual-embedding

# 2. gopedia 이미지 재빌드 (contextual embedding 코드 반영)
sudo docker compose -f docker-compose.dev.yml build gopedia

# 3. Qdrant 컬렉션 초기화 (전체 재임베딩 필요)
curl -X DELETE "http://127.0.0.1:6333/collections/gopedia_markdown"
sudo docker restart gopedia-local-gopedia-1

# 4. 전체 재인제스트
for proj in gopedia neunexus osteon taxon; do
  GOPEDIA_PHLOEM_GRPC_ADDR=localhost:50051 GOPEDIA_VERSION=v0.4.0 \
    .venv/bin/python -m property.root_props.run /neunexus/geneso/universitas/$proj
done

# 5. Architecture Blueprint 인제스트 (IMP-10)
GOPEDIA_PHLOEM_GRPC_ADDR=localhost:50051 GOPEDIA_VERSION=v0.4.0 \
  .venv/bin/python -m property.root_props.run \
  /neunexus/geneso/universitas/Universitas_System_Architecture_Blueprint.md
GOPEDIA_PHLOEM_GRPC_ADDR=localhost:50051 GOPEDIA_VERSION=v0.4.0 \
  .venv/bin/python -m property.root_props.run \
  /neunexus/geneso/universitas/docs/arch/bio_inspired_blueprint.md

# 6. gardener_gopedia IR 평가
cd /neunexus/gardener_gopedia
DATASET_ID=$(curl -s -X POST http://127.0.0.1:18880/datasets \
  -H "Content-Type: application/json" \
  -d @dataset/universitas_eval_bronze.json | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")
curl -s -X POST "http://127.0.0.1:18880/datasets/${DATASET_ID}/resolve-qrels?force=true"
RUN_ID=$(curl -s -X POST http://127.0.0.1:18880/runs \
  -H "Content-Type: application/json" \
  -d "{\"dataset_id\":\"${DATASET_ID}\",\"top_k\":10,\"index_version\":\"v0.4.0\"}" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")
sleep 120
curl -s "http://127.0.0.1:18880/runs/${RUN_ID}/metrics" | python3 -c "
import sys,json
for m in json.load(sys.stdin):
    if m['scope']=='aggregate' and not m['metric_name'].startswith('summary'):
        print(f'{m[\"metric_name\"]}: {m[\"value\"]:.4f}')
"
```
