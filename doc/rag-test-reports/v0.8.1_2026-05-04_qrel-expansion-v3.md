## P@3 분석

P@3 = 0.389 = 35/90 (30 queries × 3)

- 5개 쿼리: secondary가 top-3 내 검색 → P@3_i = 2/3 기여
- 25개 쿼리: primary만 top-3 → P@3_i = 1/3 기여

### 목표 P@3 ≥ 0.60 달성 조건

현재 retrieval 기준:
- qrel 수만으로는 달성 불가. secondary가 top-3에 들어오지 않음
- 추가 작업 필요:

1. **인접 청크 qrel 추가**: neighbor_window로 top-3에 실제 들어오는 l3_id 파악 후 qrel 등록
2. **쿼리당 3개 qrel**: 같은 섹션(l2_id) 내 청크 2개 이상 추가
3. **gardener 상세 결과 API 활용**: top2/top3 l3_id 추출 기능 필요 (현재 top1만 제공)
