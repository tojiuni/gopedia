## 분석: FTS 하이브리드 실패 원인

### 1. OR-query 광범위 매칭 문제

```
질의: "Osteon 00 Overview 레이어 구조 코드블록에서 베이스 OS 한 줄로 쓰인 값은?
       (Ubuntu 24.04 LTS (Base OS), OpenStack Dalmatian 윗줄)"

추출 기술 토큰: 'osteon' | '00' | 'overview' | 'ubuntu' | '04' | 'lts' |
                'base' | 'os' | 'openstack' | 'dalmatian'

FTS 매칭 건수: 188개  ← OpenStack 가이드 전체에 분산
```

"openstack", "ubuntu", "kubernetes", "kolla-ansible" 등의 기술 용어들이 corpus 전반에 분포하기 때문에 OR-query 한 번으로 100~200+ 청크가 매칭됨. 이 문서들이 RRF 융합 시 correct dense 결과와 동등하게 경쟁.

### 2. alpha=0.5 균등 가중치 — 수식 분석

RRF 점수: `score = alpha/(k + rank_dense) + (1-alpha)/(k + rank_sparse)`  
(k=60, alpha=0.5)

| 문서 유형 | 계산 | 점수 |
|---------|------|------|
| primary qrel (dense rank 1) | 0.5 / (60+1) | **0.00820** |
| FTS-only 오류 문서 (FTS rank 1) | 0.5 / (60+1) | **0.00820** |
| FTS-only 오류 문서 (FTS rank 2) | 0.5 / (60+2) | 0.00806 |
| FTS-only 오류 문서 (FTS rank 3) | 0.5 / (60+3) | 0.00794 |

→ 188개 FTS 매칭 중 상위 4개가 primary qrel(dense rank 1)과 동점 이상 → primary가 rank 5+ 밀려남.

### 3. Recall@5=0 회귀 — 3개 쿼리 원인

| 쿼리 | v0.11.0 Recall@5 | v0.12.0b Recall@5 | 원인 |
|------|-----------------|------------------|------|
| osteon_01_base_os | 1.0 | **0.0** | "ubuntu/openstack/dalmatian" → 188건 매칭, primary 이탈 |
| osteon_07_k8s_install_method | 0.5 | **0.0** | "kubernetes/openstack/kubeadm" → 141건 매칭, primary 이탈 |
| osteon_19_kolla_ansible_version | 1.0 | **0.0** | "kolla-ansible/dalmatian/pip" → 97건 매칭, primary 이탈 |

### 4. alpha 딜레마 — 구조적 해결 불가

FTS-only MISS 문서를 top-5에 진입시키려면 낮은 alpha(FTS 가중치 증가) 필요:

```
FTS rank 1 (MISS 문서) > dense rank 30 (lowest candidate) 조건:
  (1-alpha)/(k+1) > alpha/(k+30)
  → alpha < 0.5 (FTS에 50% 이상 가중치)
```

그러나 alpha < 0.5에서는 FTS 상위 문서들이 dense rank 1 primary qrel도 밀어냄.  
**어떤 alpha에서도 MISS 회수와 primary qrel 보존을 동시에 달성 불가.**

## FTS 단독 MISS 회수율 (인프라 효과 검증)

FTS 자체 성능은 개선됨 — 문제는 RRF 융합 방식:

| MISS qrel | FTS 순위 (limit=60) | 비고 |
|-----------|-------------------|------|
| osteon_05 `2082912a` | rank 25 ✅ | "ost-stor-01" keyword hit |
| osteon_07 `d26dbc1b` | rank 104 ❌ | `\| 설치 방식 \| kubeadm \|` — 너무 짧은 청크 |
| osteon_13 `eecbe227` | rank 13 ✅ | Floating IP 두 번째 위치 |
| osteon_21 `880deb10` | rank 72 ❌ | /etc/kolla/proxysql — 의미적 불일치 |
| osteon_24 `4ef0b53f` | rank 2 ✅ | "k8s-subnet/10.0.0.0" keyword hit |
| osteon_27 `1146ddf3` | rank 11 ✅ | clouds.yaml 두 번째 위치 |

→ FTS 인프라 자체는 6개 중 4개를 회수할 수 있으나, RRF 융합 시 오류 문서 유입으로 전체 precision 붕괴.
