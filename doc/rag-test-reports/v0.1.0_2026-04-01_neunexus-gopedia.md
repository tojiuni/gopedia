## 3. 판정

### 3-1. 항목별 합격 기준 (0.50 미만 = 실패)

| 영역 | 쿼리 수 | ≥ 0.65 | 0.50~0.64 | < 0.50 | 판정 |
|------|---------|--------|-----------|--------|------|
| neunexus | 6 | 5 | 1 | 0 | ✅ **PASS** |
| gopedia universitas | 8 | 3 | 4 | 1 | ⚠️ **PARTIAL** |

- **neunexus**: 구조가 명확한 인프라 문서라 score가 높음. N1(traefik label, 0.564)은 쿼리가 범용적이어서 낮음.
- **gopedia**: 개념 문서 특성상 score 분산이 넓음. G6(smart sink, 0.474)는 쿼리가 내부 용어 위주여서 낮게 나옴.

### 3-2. 발견된 이슈

| 이슈 | 심각도 | 설명 |
|------|--------|------|
| **중복 인제스트** | 🟡 Medium | `Gopedia Feature Guide`, `Wiki KR/EN` 등이 project_id 5와 14 양쪽에 중복 저장. Qdrant에도 중복 벡터 생성됨. 동일 content에 score가 2개씩 나타남 |
| **Generic title** | 🟡 Medium | `Readme`, `Skill`, `Index` 등 generic title로 인해 검색 결과에서 출처 파악이 어려움. `source_path` 또는 `project_name` 이 결과에 없음 |
| **Low score: smart sink** | 🟢 Low | `gopedia smart sink routing strategy` 쿼리 0.474. 한국어 기술 용어(`Smart Sink`, `파티션`)가 영어 쿼리와 임베딩 거리가 있음 |
| **코드 파일 미인제스트** | 🟢 Low | neunexus / gopedia universitas의 실제 소스 코드 파일(있을 경우)은 이번 테스트에 포함되지 않음 |

## 5. 재현 방법

```bash
# 1. 스택 시작 (Colima 기준)
docker-compose -f docker-compose.dev.yml --env-file .env --profile app up -d

# 2. 헬스체크
curl -s http://127.0.0.1:18787/api/health

# 3. 인제스트 (neunexus 전체)
GOPEDIA_PHLOEM_GRPC_ADDR=localhost:50051 \
POSTGRES_HOST=127.0.0.1 POSTGRES_USER=admin_gopedia \
POSTGRES_PASSWORD=changeme_local_only POSTGRES_DB=gopedia \
python -m property.root_props.run \
  /Users/dong-hoshin/Documents/dev/geneso/universitas/neunexus

# 4. 인제스트 (gopedia universitas 전체)
python -m property.root_props.run \
  /Users/dong-hoshin/Documents/dev/geneso/universitas/gopedia

# 5. 검색 테스트
curl -s "http://127.0.0.1:18787/api/search?q=traefik+port+80+443&format=json&detail=standard"
curl -s "http://127.0.0.1:18787/api/search?q=gopedia+L1+L2+L3&format=json&detail=standard"
```

| High | **중복 인제스트 방지**: 동일 content_hash L1이 이미 있을 경우 project 연결만 업데이트 | 중복 문서 |
